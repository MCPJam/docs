This page provides a complete reference for all CLI commands and options available in MCPJam Inspector.

## Command Syntax

```bash
npx @mcpjam/inspector@latest [OPTIONS] [SERVER_COMMAND] [SERVER_ARGS...]
```

## Global Options

### `--port <number>`

Specify the port for the Inspector web interface.

```bash
npx @mcpjam/inspector@latest --port 4000
```

- **Default**: `6274` (or next available port)
- **Range**: `1024-65535`
- **Behavior**: If the specified port is unavailable, Inspector will exit with an error

### `--ollama <model>`

Launch Inspector with automatic Ollama integration.

```bash
npx @mcpjam/inspector@latest --ollama llama3.2
```

- **Prerequisites**: Ollama must be installed
- **Behavior**: 
  1. Verifies Ollama installation
  2. Opens terminal to pull specified model
  3. Starts Ollama server
  4. Configures Inspector with Ollama provider

**Supported Models**:
- `llama3.2` - General purpose model
- `codellama` - Code-focused model  
- `mistral` - Efficient European AI model
- `deepseek-r1` - Reasoning specialized model
- Any valid Ollama model name

### `-e <KEY=VALUE>`

Set environment variables for the MCP server.

```bash
# Single variable
npx @mcpjam/inspector@latest -e DEBUG=true python server.py

# Multiple variables  
npx @mcpjam/inspector@latest \
  -e API_KEY=secret \
  -e LOG_LEVEL=debug \
  -e DATABASE_URL=sqlite:///test.db \
  python server.py
```

- **Format**: `KEY=VALUE` (no spaces around `=`)
- **Multiple**: Use multiple `-e` flags for multiple variables
- **Scoping**: Variables are passed only to the MCP server process

### `--env-file <path>`

Load environment variables from a file.

```bash
npx @mcpjam/inspector@latest --env-file .env python server.py
```

- **Format**: Standard `.env` file format
- **Example `.env` file**:
  ```bash
  API_KEY=your-secret-key
  DEBUG=true
  LOG_LEVEL=info
  DATABASE_URL=postgresql://user:pass@localhost/db
  ```

### `--help`

Display help information.

```bash
npx @mcpjam/inspector@latest --help
```

### `--version`

Display version information.

```bash
npx @mcpjam/inspector@latest --version
```

## Server Connection Commands

### STDIO Server Connection

Connect to MCP servers using standard input/output.

```bash
# Python servers
npx @mcpjam/inspector@latest python /path/to/server.py
npx @mcpjam/inspector@latest uv run fastmcp run /path/to/server.py
npx @mcpjam/inspector@latest ./venv/bin/python server.py

# Node.js servers
npx @mcpjam/inspector@latest node /path/to/server.js
npx @mcpjam/inspector@latest npx -y @package/mcp-server

# Compiled binaries
npx @mcpjam/inspector@latest ./dist/server
npx @mcpjam/inspector@latest /usr/local/bin/mcp-server
```

**Requirements**:
- Server command must be executable
- Use absolute paths for reliability
- Server must implement MCP STDIO protocol

### Command Arguments

Pass arguments to your MCP server:

```bash
# Arguments are passed through to the server
npx @mcpjam/inspector@latest python server.py --config prod.json --debug
npx @mcpjam/inspector@latest node server.js --port 8080 --env production
```

### Argument Separation

Use `--` to explicitly separate Inspector options from server arguments:

```bash
npx @mcpjam/inspector@latest --port 3001 -- python server.py --port 8080
#                            ^^^^^^^^^^ ^^ ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
#                            Inspector  │  Server command and args
#                            options    └─ Explicit separator
```

This is useful when:
- Server uses conflicting option names (e.g., `--port`)
- You want to be explicit about argument ownership
- Debugging argument parsing issues

## Environment and Configuration

### Working Directory

The MCP server process inherits the current working directory by default:

```bash
cd /path/to/my/project
npx @mcpjam/inspector@latest python server.py
# Server runs with CWD = /path/to/my/project
```

### Process Management

Inspector manages the MCP server process lifecycle:

- **Startup**: Spawns server process when Inspector starts
- **Monitoring**: Monitors process health and restarts if needed
- **Shutdown**: Cleanly terminates server when Inspector exits
- **Logging**: Captures server stdout/stderr for debugging

### Signal Handling

Inspector handles system signals gracefully:

```bash
# Ctrl+C (SIGINT) - Graceful shutdown
# SIGTERM - Graceful shutdown  
# SIGKILL - Force kill (not recommended)
```

## Advanced Options

### Debug Mode

Enable verbose logging for troubleshooting:

```bash
DEBUG=* npx @mcpjam/inspector@latest python server.py
```

**Available Debug Namespaces**:
- `mcpjam:server` - Server process management
- `mcpjam:mcp` - MCP protocol messages
- `mcpjam:http` - HTTP server logs
- `mcpjam:*` - All debug output

### Production Mode

Run in production mode with optimizations:

```bash
NODE_ENV=production npx @mcpjam/inspector@latest python server.py
```

**Production Optimizations**:
- Reduced logging verbosity
- Optimized asset serving
- Enhanced error handling
- Performance monitoring

### Resource Limits

Set resource limits for the MCP server process:

```bash
# Limit memory usage (Linux/macOS)
ulimit -m 512000  # 500MB limit
npx @mcpjam/inspector@latest python server.py

# Limit CPU time
ulimit -t 300     # 5 minute CPU limit
npx @mcpjam/inspector@latest python server.py
```

## Exit Codes

Inspector returns specific exit codes for different scenarios:

| Exit Code | Meaning |
|-----------|---------|
| `0` | Success - Normal shutdown |
| `1` | General error - Unknown or unexpected error |
| `2` | Invalid arguments - Command line parsing error |
| `3` | Server error - MCP server failed to start or crashed |
| `4` | Port error - Specified port unavailable |
| `5` | Permission error - Insufficient permissions |
| `6` | Not found - Command or file not found |

### Usage in Scripts

```bash
#!/bin/bash
npx @mcpjam/inspector@latest python server.py
exit_code=$?

case $exit_code in
  0) echo "Inspector shut down normally" ;;
  3) echo "MCP server error - check server logs" ;;
  4) echo "Port conflict - try different port" ;;
  *) echo "Unknown error (exit code: $exit_code)" ;;
esac

exit $exit_code
```

## Platform-Specific Considerations

### Windows

```cmd
REM Use Windows path format
npx @mcpjam/inspector@latest python C:\path\to\server.py

REM Use forward slashes or escaped backslashes
npx @mcpjam/inspector@latest python C:/path/to/server.py
npx @mcpjam/inspector@latest python C:\\path\\to\\server.py
```

### macOS

```bash
# Use absolute paths for reliability
npx @mcpjam/inspector@latest python /Users/username/project/server.py

# Handle spaces in paths
npx @mcpjam/inspector@latest python "/Users/username/My Project/server.py"
```

### Linux

```bash
# Standard Unix paths
npx @mcpjam/inspector@latest python /home/user/project/server.py

# Use which to find interpreter
npx @mcpjam/inspector@latest $(which python3) server.py
```

## Configuration Files

### Inspector Configuration

Inspector looks for configuration in these locations:

1. `./mcpjam.config.json` (current directory)
2. `~/.config/mcpjam/config.json` (user config)
3. `/etc/mcpjam/config.json` (system config)

**Example Configuration**:
```json
{
  "defaultPort": 3001,
  "servers": [
    {
      "name": "Development Server",
      "command": "python",
      "args": ["/path/to/server.py"],
      "env": {
        "DEBUG": "true"
      }
    }
  ],
  "ollama": {
    "defaultModel": "llama3.2",
    "baseUrl": "http://localhost:11434"
  }
}
```

### Environment Variable Priority

Environment variables are resolved in this order (highest to lowest priority):

1. CLI `-e` flags
2. `--env-file` contents
3. System environment variables
4. Configuration file `env` section
5. Inspector defaults

## Troubleshooting Commands

### Connectivity Testing

```bash
# Test if command exists and is executable
which python
python --version

# Test server command manually
python /path/to/server.py

# Check port availability
netstat -an | grep :3001
lsof -i :3001
```

### Process Monitoring

```bash
# Monitor Inspector and server processes
ps aux | grep inspector
ps aux | grep python  # or your server process

# Monitor resource usage
top -p $(pgrep -f inspector)
htop -p $(pgrep -f server.py)
```

### Log Analysis

```bash
# Enable detailed logging
DEBUG=mcpjam:* npx @mcpjam/inspector@latest python server.py 2>&1 | tee inspector.log

# Filter specific log levels
DEBUG=mcpjam:* npx @mcpjam/inspector@latest python server.py 2>&1 | grep ERROR

# Tail logs in real-time
DEBUG=mcpjam:* npx @mcpjam/inspector@latest python server.py 2>&1 | tail -f
```

## Security Considerations

### Safe Environment Variables

```bash
# ❌ Avoid exposing secrets in command line
npx @mcpjam/inspector@latest -e SECRET_KEY=mysecret python server.py

# ✅ Use environment file instead
echo "SECRET_KEY=mysecret" > .env
npx @mcpjam/inspector@latest --env-file .env python server.py
```

### File Permissions

```bash
# Ensure server script is not world-writable
chmod 755 server.py  # rwxr-xr-x

# Protect configuration files
chmod 600 .env       # rw-------
chmod 600 config.json
```

### Network Security

```bash
# Bind to localhost only (default)
npx @mcpjam/inspector@latest --port 3001 python server.py

# For production, use reverse proxy instead of exposing directly
# nginx, Apache, or similar should handle external access
```
