The MCPJam Inspector CLI provides a powerful command-line interface for launching Inspector with various configurations and connecting to MCP servers directly. This is the fastest way to get started testing your MCP servers.

## Installation

The CLI is distributed as an NPM package and can be run directly with NPX:

```bash
# Run latest version (recommended)
npx @mcpjam/inspector@latest

# Install globally (optional)
npm install -g @mcpjam/inspector
inspector
```

<Note>
Using NPX ensures you always run the latest version without needing to manage updates.
</Note>

## Basic Usage

### Quick Start

Launch Inspector on the default port (3001):

```bash
npx @mcpjam/inspector@latest
```

This starts the Inspector web interface at `http://localhost:3001`.

### Custom Port

Specify a custom port for the Inspector interface:

```bash
npx @mcpjam/inspector@latest --port 4000
```

The application will be available at `http://localhost:4000`.

## Direct MCP Server Connection

The most powerful feature of the CLI is the ability to launch Inspector with an automatic connection to your MCP server:

### Python Servers

```bash
# FastMCP server
npx @mcpjam/inspector@latest uv run fastmcp run /Users/dev/server.py

# Standard Python server
npx @mcpjam/inspector@latest python /Users/dev/server.py

# With virtual environment
npx @mcpjam/inspector@latest ./venv/bin/python server.py

# With arguments
npx @mcpjam/inspector@latest python server.py --debug --config prod.json
```

### Node.js Servers

```bash
# Direct execution
npx @mcpjam/inspector@latest node /Users/dev/server.js

# NPX package execution
npx @mcpjam/inspector@latest npx -y your-mcp-package

# With arguments
npx @mcpjam/inspector@latest node server.js --port 8080 --env production
```

### Executable Files

```bash
# Any executable MCP server
npx @mcpjam/inspector@latest /path/to/mcp-server --options

# Compiled binaries
npx @mcpjam/inspector@latest ./dist/server --config config.toml
```

<Warning>
**Absolute Paths Required**: Always use absolute file paths when specifying server commands to avoid path resolution issues.
</Warning>

## Ollama Integration

Inspector provides seamless integration with Ollama for local AI model testing:

### Quick Ollama Setup

```bash
# Launch Inspector with Ollama model
npx @mcpjam/inspector@latest --ollama llama3.2
```

This command:
1. Checks if Ollama is installed
2. Opens a terminal window to pull the specified model
3. Starts the Ollama server
4. Launches Inspector with Ollama pre-configured

### Supported Models

```bash
# Popular models
npx @mcpjam/inspector@latest --ollama llama3.2      # General purpose
npx @mcpjam/inspector@latest --ollama codellama     # Code-focused
npx @mcpjam/inspector@latest --ollama mistral       # Efficient European model
npx @mcpjam/inspector@latest --ollama deepseek-r1   # Reasoning specialized
```

### Prerequisites for Ollama

Ensure Ollama is installed before using `--ollama`:

```bash
# Check if Ollama is installed
ollama --version

# Install Ollama if needed
# Visit: https://ollama.ai/download
```

## Environment Configuration

### Environment Variables

Pass environment variables to your MCP server:

```bash
# Single environment variable
npx @mcpjam/inspector@latest -e DEBUG=true python server.py

# Multiple environment variables
npx @mcpjam/inspector@latest \
  -e API_KEY=your-key \
  -e LOG_LEVEL=debug \
  -e DATABASE_URL=sqlite:///test.db \
  python server.py
```

### Configuration Files

Use configuration files for complex setups:

```bash
# Load environment from .env file
npx @mcpjam/inspector@latest --env-file .env python server.py

# Specify custom config
npx @mcpjam/inspector@latest python server.py --config custom-config.json
```

## Advanced Usage Patterns

### Development Workflow

```bash
# Development with auto-reload
npx @mcpjam/inspector@latest \
  --port 3001 \
  -e NODE_ENV=development \
  -e DEBUG=true \
  nodemon server.js
```

### Testing Multiple Servers

```bash
# Launch separate instances for different servers
npx @mcpjam/inspector@latest --port 3001 python server1.py &
npx @mcpjam/inspector@latest --port 3002 python server2.py &
npx @mcpjam/inspector@latest --port 3003 node server3.js &
```

### CI/CD Integration

```bash
# Headless testing mode (future feature)
npx @mcpjam/inspector@latest \
  --headless \
  --validate \
  --output report.json \
  python server.py
```

## Command Structure

### General Syntax

```bash
npx @mcpjam/inspector@latest [OPTIONS] [SERVER_COMMAND] [SERVER_ARGS...]
```

### Arguments Processing

The CLI processes arguments in this order:

1. **Inspector Options**: `--port`, `--ollama`, `-e`, etc.
2. **Separator** (optional): `--` to explicitly separate Inspector from server args
3. **Server Command**: The command to execute your MCP server
4. **Server Arguments**: All arguments passed to your server

### Example with Separator

```bash
# Explicit separation of arguments
npx @mcpjam/inspector@latest --port 4000 -- python server.py --debug --config prod.json
#                            ^^^^^^^^^^ ^^ ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
#                            Inspector  │  Server command and arguments
#                            options    └─ Separator
```

## Debugging and Troubleshooting

### Verbose Output

Enable detailed logging for debugging:

```bash
# Enable debug output
DEBUG=* npx @mcpjam/inspector@latest python server.py

# Or use environment variable
export DEBUG=mcpjam:*
npx @mcpjam/inspector@latest python server.py
```

### Common Issues

<Accordion title="Command not found">
  **Error**: `Command 'python' not found`
  
  **Solutions**:
  - Use absolute path: `/usr/bin/python3 server.py`
  - Check PATH: `echo $PATH`  
  - Verify command exists: `which python3`
</Accordion>

<Accordion title="Permission denied">
  **Error**: `EACCES: permission denied`
  
  **Solutions**:
  - Make file executable: `chmod +x server.py`
  - Use interpreter: `python server.py` instead of `./server.py`
</Accordion>

<Accordion title="Port already in use">
  **Error**: `EADDRINUSE: address already in use`
  
  **Solutions**:
  - Use different port: `--port 4000`
  - Kill existing process: `lsof -ti:3001 | xargs kill`
  - Let Inspector find available port automatically
</Accordion>

## Integration Examples

### Docker Integration

```bash
# Launch Inspector pointing to dockerized MCP server
docker run -d -p 8080:8080 my-mcp-server
npx @mcpjam/inspector@latest --port 3001
# Then connect to http://localhost:8080 in the UI
```

### Development Scripts

Create npm scripts for common workflows:

```json
{
  "scripts": {
    "dev": "npx @mcpjam/inspector@latest python src/server.py",
    "dev:debug": "npx @mcpjam/inspector@latest -e DEBUG=true python src/server.py",
    "test:mcp": "npx @mcpjam/inspector@latest --port 3001 python tests/test_server.py"
  }
}
```

### Shell Aliases

Create convenient shell aliases:

```bash
# Add to .bashrc or .zshrc
alias mcpi="npx @mcpjam/inspector@latest"
alias mcpi-py="npx @mcpjam/inspector@latest python"
alias mcpi-node="npx @mcpjam/inspector@latest node"

# Usage
mcpi-py /path/to/server.py
mcpi-node /path/to/server.js
```

## Performance Considerations

### Startup Time

- **NPX overhead**: ~2-3 seconds for package download/cache check
- **Server startup**: Depends on your MCP server implementation  
- **Inspector initialization**: ~1-2 seconds

### Resource Usage

- **Memory**: ~50-100MB for Inspector process
- **CPU**: Minimal when idle, scales with MCP server activity
- **Network**: Only local traffic unless using remote MCP servers
