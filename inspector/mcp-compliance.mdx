MCPJam Inspector provides comprehensive validation of your MCP server implementation against the complete Model Context Protocol specification. This ensures your server works correctly with all MCP-compatible AI applications and clients.

## What Gets Validated

### Tools Validation
Inspector thoroughly tests your MCP tools to ensure they meet specification requirements:

- **Schema Validation**: Verifies tool definitions include required fields (name, description)
- **Parameter Validation**: Checks parameter types, constraints, and required/optional fields
- **Input/Output Format**: Ensures proper JSON schema compliance for tool inputs and outputs
- **Error Handling**: Tests how your tools handle invalid inputs and edge cases

### Resources Validation
Complete validation of your MCP resources:

- **Resource Schema**: Verifies resource definitions and metadata
- **Content Types**: Validates supported MIME types and content formats
- **URI Templates**: Checks URI template syntax and variable substitution
- **Access Control**: Tests resource permissions and access patterns

### Prompts & Elicitation
Validation of conversational capabilities:

- **Prompt Templates**: Verifies prompt structure and variable interpolation
- **Context Handling**: Tests context window management and token limits
- **Response Formatting**: Ensures proper response structure and metadata
- **Streaming Support**: Validates streaming response capabilities

### OAuth 2.0 Authentication
Complete OAuth 2.0 flow testing:

- **Authorization Flow**: Tests the complete authorization code flow
- **Token Management**: Validates token request, refresh, and revocation
- **Scope Handling**: Verifies proper scope validation and restrictions
- **Error Responses**: Ensures proper OAuth error handling and responses

## Common Compliance Issues

### Tool Definition Problems
<Warning>
  **Missing Required Fields**: Tools must include `name` and `description` fields
</Warning>

```json
// ❌ Invalid - missing description
{
  "name": "get_weather"
}

// ✅ Valid - includes required fields  
{
  "name": "get_weather",
  "description": "Get current weather for a location",
  "inputSchema": {
    "type": "object",
    "properties": {
      "location": {"type": "string"}
    },
    "required": ["location"]
  }
}
```

### Parameter Type Mismatches
<Warning>
  **Type Validation**: Ensure parameter types match their schema definitions
</Warning>

```json
// ❌ Invalid - expects string but receives number
{
  "location": 12345
}

// ✅ Valid - correct string type
{
  "location": "San Francisco, CA"
}
```

### OAuth Configuration Issues
<Warning>
  **Redirect URI Mismatch**: OAuth redirect URIs must exactly match registered values
</Warning>

## Best Practices

### 1. Start with Schema Design
Design your tool and resource schemas carefully:

```json
{
  "inputSchema": {
    "type": "object",
    "properties": {
      "query": {
        "type": "string",
        "description": "Search query",
        "minLength": 1,
        "maxLength": 500
      }
    },
    "required": ["query"],
    "additionalProperties": false
  }
}
```

### 2. Implement Proper Error Handling
Provide clear error messages for invalid inputs:

```python
def validate_input(data):
    if not data.get('query'):
        raise ValueError("Query parameter is required")
    
    if len(data['query']) > 500:
        raise ValueError("Query too long (max 500 characters)")
        
    return True
```

### 3. Test Edge Cases
Use Inspector to test various edge cases:

- Empty or null parameters
- Extremely long inputs
- Invalid data types
- Network timeouts
- Authentication failures

### 4. Monitor Compliance Continuously
Set up automated compliance monitoring:

```bash
# Run compliance check in CI/CD
npx @mcpjam/inspector@latest --validate --server-command "python server.py" --output compliance-report.json
```

## Compliance Levels

Inspector categorizes compliance into different levels:

### ✅ Fully Compliant
- All required fields present
- Proper type validation
- Complete error handling
- OAuth flows working correctly

### ⚠️ Warning Level
- Minor specification deviations
- Missing optional fields
- Suboptimal error messages
- Performance concerns

### ❌ Non-Compliant
- Missing required fields
- Type validation failures
- Broken authentication flows
- Protocol violations
